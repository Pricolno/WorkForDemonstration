> [лин модели](https://education.yandex.ru/handbook/ml/article/linear-models)

* **Линейная регрессия**: $w^T x + w_0$, с MSE, MAE, MAPE

есть точная формула для $L(f, X, y) = \|y - Xw\|_2^2$ \
$L_{w}^{'} = -y^T Xw - (Xw)^Ty + (Xw)^T(Xw) = -2X^T y + 2 X^T X w = 0$ \

$w = (X^TX)^{-1} X^T y $
> долго обращать матрицу, иногда нельзя, лучше градиентным спуском!

* классифицацию решает линейная регрессия $w^T + w_0 > 0$, а есть логистическая!! 

M = y<w, x>, {-1, 1} - метки классов \
L(y, w, x) =  max(0, -M) \
L(y, w, x) =  max(0,1 -M) [SVM]

* **Логистическая регрессия**
Ключевая идея: хотим предсказывать [0, 1], а дано $[-\infty, \infty ]$

Подходит $<w, x> =\ln (\frac{p}{1 - p})$, $p = \frac{1}{1 + e^{-<w, x>}} = \sigma(<w, x>)$ 

Мы зафиксировали модель, теперь в ней нужно найти оптимальные веса. Что минизировать? (почему кросс энтропию?) 

Баесовский вывод, а именно пусть $P(y | x, w) = p^y (1 - p)^{1 - y}$ Из распределения бернули (мы предсказываем: $w^T x= p$), в итоге максимизируем кросс энтропию!

* **Многоклассовая**
    * one-vs-all: обучаем n классификаторов, один против всех и предсказываем вероятность класс, с самым увереным классификатором. 
    * one-vs-all: обучаем $n^2$ классификаторов, предсказывать класс i,j. тогда $a(x) = \argmax_k \sum_i \sum_j a_{i, j}(x) = k$ т.e. класс, который собрал больше всех попарных голосов тот и выйграл

    * можно сразу, по n логитам, применить soft-max  [фикс модель, и берем за вероятность, тогда правдоподобие превращается  в многокласовую кросс энтропию]

>Оценка максимального правдоподобия, как она связана с регрессией и классификацией

Модель: $w^T x = \mu$ \
Вероятностная модель: $P(\mu|x, w) = N(\mu, \sigma)$ 

$N(\mu, \sigma) = \frac{1}{\sigma \sqrt{2\pi}} e^{\frac{-1}{2}(\frac{z - \mu}{\sigma})^2}$

$\ln P(\mu = \mu_i |X, w) = \ln \prod_i P(\mu_i| x_i, w) = \sum_i -\ln (\sigma \sqrt{2\pi} ) - \frac{1}{2}\frac{(z_i - \mu_i )^2}{\sigma^2}$ -> max

$w = \argmin_w MSE(z_i, w^T x)$

Классификационные модели **деляться** на 
* $P(Y| X)$ - дискриминативные (лог регрессия)
* $P(X, Y)$ = P(X|Y) P(Y) - генеративные, Теорема Баеса $P(Y_i|X) =  \frac{P(Y_i) P(X|Y_i)}{\sum_i P(Y_i) P(X|Y_i)}$ тогда $a(x) = \argmax_y P(Y|X) = \argmax_y P(Y_i) P(X| Y_i)$

**Gaussian discriminant analysis (GDA)**

Каждый класс имеет мнргомерное нормальное распределние. Если ковариционная матрица одинаковая, то разделяющая поверхность гиперплоскоссть порядка 1, иначе 2

**Linear Discriminant Analysis (LDA)**
 

**Метод наивного байеса**

Считаем, что условные фичи независимы $P(X_1, X_2| Y) = P(X_1|Y) P(X_2| Y)$ - дальше оценивание параметров для одномерных распределний [получаем GDA с диагональной ков матрицей]

[если модель непараметрическая, то можно завести ядро и плотность в точке определять по нему]

**Цель бустинга**:  успеньшить смещение, тогда для деревьев подходят пни. Итерационно следущая модель учит разницу между пресказанием суммы предыдущих. 

**Формально**: нужен лосс L(y, a(x)) = MSE, тогда каждый следующий алгорит предсказывает $s_i = y_i - a_n(x_i)$, оказывается при MSE модели предсказывают антиградиет $s_i^k -> g_i^k = \frac{\partial L(y_i, z)}{\partial z} |_{z = a_k(x_i)}$ 

тогда мы просто движемся по антигадиенту лосс функции $a_1(x_1), ..., a_1(x_n) -> a_2(x_1), ..., a_2(x_n) -> a(x_1), ..., a(x_n)$

# 26
>Методы оптимизации в глубоком обучении. Градиентный спуск, SGD, его улучшения: Momentum, RMSProp, Adam. [Оптимизации в ML](https://education.yandex.ru/handbook/ml/article/optimizaciya-v-ml)

Чтобы найти локальный минимум нужно двигаться в сторону антиградиента! 

* **GD**: $x_{k + 1} = x_k - lr \partial f(x_k)$
* **SGD**: GD по батчу
* **Momentum**: добавляется инерция $x_{k + 1} = x_k - lr \partial f(x_k) + \beta (x_k - x_{k - 1})$

Функцию ошибку считать сложно, нужно придумать адаптивные алгоритмы, не только пользоваться градиентами

* **AdaGrad**: 
    * $G_{k + 1} = G_k + (\partial f(x_k))^2$
    * $x_{k + 1} = x_k + \frac{\alpha}{\sqrt{G_{k + 1} + \varepsilon}} \partial f(x_k)$
* **RMSProp**: добавляется скользящее окно
    * $G_{k + 1} = \gamma G_k + (1 - \gamma)(\partial f(x_k))^2$
    * $x_{k + 1} = x_k + \frac{\alpha}{\sqrt{G_{k + 1} + \varepsilon}} \partial f(x_k)$

* **Adam** ADAptive Momentum
    *  $v_{k + 1} = \beta_1 v_k + (1 - \beta_1) \partial f(x_k)$
    * $G_{k + 1} = \beta_2 G_k + (1 - \beta_2)(\partial f(x_k))^2$
    * $x_{k + 1} = x_k - \frac{\alpha}{\sqrt{G_{k + 1} + \varepsilon}} v_{k + 1}$
    * замечание: берут $\beta_1 =0.9, \beta_2 = 0.99,\varepsilon =1e-8  $

# 30
>Операция свертки. Свёрточный слой, обратное распространение через него. Гиперпараметры свёрток. Сравнение 1x1 свёрток и полносвязных слоёв. Пулинг: max/average. [тык](https://education.yandex.ru/handbook/ml/article/svyortochnye-nejroseti)

Используются для картинок. Так важно локальные признаки. Хотим быть инвариантны к паралельным переносам.

**Свертка** - слой проходится ядром (обычно 3*3 обучаемых весов) по картинке. 

**receptive field** - видимость области исходной картинки

**Гиперпараметры свёрток** 
* **размер ядра**
* **max pooling**, **average pooling** - понижение размерности по непересекающимся окнам
* **stride** - увелечение шага свертки
* **padding** - добавления краев
* **dilated convolution**, ядро разреживается
* **Global average pool** - на последних слоях усредняем каналы. 

* **Residual connection** - борьба с затуханием градиента, предыдущие слои прибавлять к будущим.

**Способы аугментации** 
* сдвиги, повороты и отражения
* добавление случайного гауссового шума
* вырезание случайно части картинки (cutout);
* перспективные преобразования
* случайное изменение оттенка\насыщщености\яркости для всей картинки;

Паралельно считам свертки из разных масшабов. Когда много накапливается размерностей, применяем 1*1 свертку. 

* **ResNet (2015)**: [152 слоя] глубокая сверточная сеть с **skip connection** [тык](https://habr.com/ru/articles/688350/) - важно! она поомгает не тоько с затуханием градиента, а и улучшает ландшафт функции ошибки - она становится менее симметричная, кол-во ложных локальных минимумов уменьшается!
