## Теория

### Алгосы и структуры данных 

**Cортировки**
> неэффективные
* Пузырек [проходимся слева направо и свапаем наибольший вправо] O(n^2)
* Сортировка вставка [пусть префикс отсорчен, то следующий элемент ищет себе место и вставляется туда] O(n^2)
* Сортировка выбором [пусть у префикса и суфикса нету инверсий, тогда минимум на суффиксе встает в конец префикса]

> эффективные

* MergeSort [делим пополам, сортим рекурсивно обе части, обьединяем слиянием] O(n ln n)

* Quicksort [выбираем pivot, раскидываем на две группы, меньшая и большая чем pivot. Рекурсивно сорти] O(n ln n)

>

**Хэш таблицы**, [habr](https://habr.com/ru/articles/509220/)

Хотим O(1) (?!) вставка, удаление, поиск 

Хотим иметь `hash: el -> int`, проблема коллизий


1) один массив, кладем элемент в ячейку с номером хеша [при коллизии можно в список неудорядоченный пихать]

2) двойное хеширование, table[`hash_1`][`hash_2`]= el [нужен идеальный хеш, при нем можно перестраивать при колизии]

3) двойное хеширование, с одним массивом: кладем в свободную ячейку h1, h1 + h2, ... , h1 + n * h2

> 

**Дерево поиска** 

* [для быстрого поиска элементов]Бинарное дерево - дерево подвешенное за вершину, где любой родитель больше левого соседа и меньша или равен правому 
[далее можно увеличивать кол-во ветвлений, так появляются B-деревья]. O(log n) - добавление/удаление/поиск

* [для быстрого нахождения минимума] Куча - дерево, подвешенное за вершину, где родители больше, чем дети. Поиск минимума O(1), вставка,удаление O(log n)

* Если смешать, получим Декартов дерево (дуча) - (x, y): x ключи, y веса [дерево поиска по x, куча по y, строится однозначно]
     
    * Если использовать вместо ключей, размер поддерева - получим декартач по неявному ключу [массив]
