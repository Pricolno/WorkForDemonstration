# Backend developer

Спроектируйте компонент для парсинга фидов (xml).

Фид содержит список элементов (`<flowers>`), описывающих
полевые цветы (`<flower>`) - название (`<title>`), описание
(`<description>`), таксоны научной классификации (`<species>`, `<genus>`
...) и ссылки на фотографии (`<photo1>`, `<photo2>` ... `<photo5>`).

Задача компонента получить фид, распарсить список, скачать
фотографии в локальное хранилище и сохранить результат своей
работы.

Он должен поддерживать возможность загрузки файла из разных
источников (внешняя ссылка, локальное хранилище), выгрузки
итоговых данных в разные хранилища (БД, облако). А так же
логирование процесса и отправку нотификаций по его завершению
(slack, e-mail).

Последовательное скачивание фотографий - бутылочное горлышко
компонента, опционально можно проработать оптимизацию этого
момента.

Ответ можно выполнить в виде UML-диаграммы в сервисе draw.io,
либо в виде верхнеуровнего наброска на РНР - интерфейсы, классы,
методы без реализации.

При необходимости добавьте комментарии, описывающие логику и
выбранные архитектурные решения.

Загрузите решение (РНР-код или ссылка на диаграмму) в
репозиторий на github/gitlab/bitbucket и приложите ссылку на него в
ответ. Не забудьте разрешить доступ.

## Интерфейс компоненты для парсинга фидов

1. `FeedParser` - класс, ответственный за получение фида в виде XML-файла и его распаковку. Содержит методы:

    - `getFeed($url)` - получает фид по заданному URL
    - `getXML($content)` - распаковывает содержимое фида и возвращает объект `SimpleXMLElement`
    - `parseFeed($xml)` - вызывает метод parseFlowers для каждого элемента `<flowers>`
    - `parseFlowers($xmlFlower)` - разбирает содержимое элемента `<flower>` и создает объект Flower
    - `parsePhotos($xmlFlower)` - разбирает фотографии, загружает их в локальное хранилище и возвращает массив ссылок в виде строк
    - `saveFlowers($flowers)` - сохраняет список объектов Flower в хранилище

2. `Flower` - класс, представляющий полевой цветок. Содержит свойства:
    - `title` (название)
    - `description` (описание)
    - `species`, `genus` (таксоны научной классификации)
    - `photos` (массив ссылок на фотографии)

3. `PhotoManager` - класс, отвечающий за загрузку фотографий и их сохранение в локальное хранилище. Содержит методы:

    - `downloadPhoto($url, $path)` - загружает фотографию по заданному URL в указанное место
    - `savePhoto($path)` - сохраняет фотографию в локальное   хранилище и возвращает путь до нее

4. `Notifier` - класс, осуществляющий отправку уведомлений о завершении работы компонента. Содержит методы:

    - `sendNotification($message)` - отправляет уведомление на заданный адрес (e-mail, Slack и т.д.)

5. `Logger` - класс, отвечающий за логирование работы компонента. Содержит методы:

    - `log($message)` - записывает сообщение в лог-файл

6. `Storage` - класс, представляющий хранилище данных. В качестве примера задана работа с MySQL. Содержит методы:

    - `connect()` - подключение к базе данных
    - `save($table, $data)` - сохранение данных в таблицу базы данных

## Добавление асинхронности

Для реализации асинхронной загрузки фотографий вместо последовательной можно использовать многопоточность. Для этого можно воспользоваться механизмом потоков, которые будут обрабатывать загрузку и сохранение фотографий параллельно друг другу. В этом случае можно реализовать следующие шаги:

1. Создать специальный класс `AsyncTaskManager`, который будет управлять запуском асинхронных задач.

2. Создать класс `AsyncPhotoDownloader`, который будет являться асинхронным потоком с загрузкой фото. Он будет принимать URL фото и путь до локального хранилища в конструкторе, и запускаться в фоновом режиме.

3. Создать класс `AsyncPhotoSaver`, который будет являться асинхронным потоком, принимающим путь до загруженного фото в локальном хранилище и сохраняющий этот файл в заданном каталоге. В конструкторе он также будет принимать путь до каталога, в который будет сохраняться файл.

4. В методе parsePhotos класса `FeedParser` создать экземпляр `AsyncTaskManager`, который будет управлять загрузкой и сохранением фотографий. Количество потоков, которые будут запускаться одновременно, можно задавать в конфигурационном файле.

5. В методе `parsePhotos` класса `FeedParser` для каждой фотографии создать экземпляр `AsyncPhotoDownloader` и добавить его в пул задач `AsyncTaskManager`. При этом можно использовать метод `addTask`, который будет отвечать за добавление в очередь новой асинхронной задачи с указанным объектом и параметрами.

6. В методе `parsePhotos` создать экземпляр `AsyncPhotoSaver` и добавить его в пул задач `AsyncTaskManager`. Предполагается, что запущенные асинхронные потоки будут автоматически записываться в списки завершенных и ошибочных задач.

7. После выполнения всех задач из пула `AsyncTaskManager` можно использовать колбек-функцию, которая будет выполняться после завершения всех задач, и продолжать выполнение методов сохранения в хранилище.

Таким образом, асинхронный подход позволит оптимизировать выполнение загрузки и сохранения фотографий в локальное хранилище, снизить время выполнения и повысить производительность.
